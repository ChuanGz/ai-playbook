# Career Project Evidence

| Learning metadata | Value |
|---|---|
| Career level | L2 Practitioner |
| Topics | career-evidence, portfolio |
| Purpose | Prove capability with reviewable outcomes |
| Importance | Important |

Strong evidence demonstrates judgment and outcomes, not activity volume.

For each project record:

1. Context, stakeholders, constraints, and baseline.
2. Personal decision scope and explicit responsibilities.
3. Alternatives considered and trade-offs made.
4. Deliverables produced and controls applied.
5. Measured quality, cost, risk, adoption, and operational outcomes.
6. Failures, corrections, and lessons reused elsewhere.
7. Links to reviewable artifacts with sensitive details removed.

Use more than one context before claiming that a competency is sustained. A repository, diagram, or generated artifact is supporting evidence only; the claim must identify the decision, contribution, consequence, and result.

## Review checklist

- [ ] Scope and ownership are explicit.
- [ ] Claims map to the target role and level.
- [ ] Baseline and acceptance criteria are visible.
- [ ] Alternatives, trade-offs, and failure modes are included.
- [ ] Outcomes are measured or uncertainty is stated.
- [ ] Team output is distinguished from personal contribution.
- [ ] Confidential data and credentials are excluded.
