This area brings the former `ai-career` People dimension into the playbook. The same repository now connects what AI systems require with what professionals must understand, decide, deliver, and verify.

## Start here

1. Use the [career roadmap](roadmap.md) to identify the next competency level.
2. Select a target from the [role map](roles.md).
3. Follow the closest [transition path](transition-paths.md) without relearning competencies already demonstrated.
4. Build a reviewable case using the [project evidence standard](project-evidence.md).

For a concrete learning project, use the [LLM, RAG, and second-brain capability checklist](llm-rag-second-brain-capability.md).

Progression requires demonstrated scope, judgment, and outcomes—not title, tenure, course completion, tool familiarity, or certificate collection.

## Principles

- Evergreen capability over fashionable tooling.
- Role clarity over title inflation.
- Evidence-based progression over activity volume.
- Domain and system outcomes over polished demos.
- Implementation literacy plus architecture and verification—not implementation avoidance.
- Vendor-neutral guidance by default.

## Boundaries

This framework defines reusable role expectations, competency levels, transition paths, and evidence standards. It does not establish an employer's job architecture, compensation, promotion policy, performance rating, hiring decision, or personal assessment record. Keep private candidate and employer-confidential evidence outside this repository.
