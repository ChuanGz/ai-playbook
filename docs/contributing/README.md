# Documentation Contribution Standards

These standards define what acceptable playbook content looks like regardless of whether it is written manually, with an editor, or with an AI-assisted workflow.

## Required reading

1. [Writing Standard](writing-standard.md)
2. [Evidence Policy](evidence-policy.md)
3. [Maintenance Policy](../../MAINTENANCE.md)
4. [Contributing](../../CONTRIBUTING.md)

## Acceptance outcome

A contribution is ready for review when a reader can identify the decision it supports, the conditions under which it applies, credible alternatives, failure behavior, evidence, and an observable pass condition.

Tools and skills may help produce the change, but they do not replace these acceptance requirements or maintainer judgment.
